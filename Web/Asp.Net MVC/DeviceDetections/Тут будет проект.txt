В этом проекте рассмотрены примеры определения информации об устройстве и разных фич клиента.
Определить тип устройства можно 2 подходами:
1) Воспользоваться облачным сервисом Wurfl. Для этого надо зарегаться на их сайте и скопировать APIKey. 
В базовом контроллере объявляем метод и на каждый реквест достаём из api информацию об устройстве.
Локальная база wurfl почему-то не корректно работает на хостинг провайдерах(при каждом запросе приложение стартует заново).
2) Более расширенный способ. Можно комбенировать несколько баз данных путём переопределения стандартного HttpCapabilitiesDefaultProvider.

Так же в проекте рассмотрен пример FeatureDetection. Мы можем определять любые настройки клиента посылая xml на View.
Данный подход следуует реализовывать если библиотека Moderniz не справляется с бизнес задачами.
